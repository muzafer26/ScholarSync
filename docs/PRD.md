# ScholarSync Product Requirements Document

## 1. Vision
To turn the search for education and career growth into an immersive, engaging journey rather than a chore.

## 2. Functional Requirements
- **Resource Categorization:** Separate flows for Courses, Opportunities, and Guides.
- **Interactive UI Components:** 
    - 3D Book with leaf-turning logic.
    - 3D Door with opening animation.
    - Animated Treasure Chest for unlocking guides.
- **Resource Discovery:** Displaying cards with metadata (Provider, Duration, Location, Category).
- **Personalization:**
    - User Authentication (Mocked).
    - Bookmarking system to save specific resources.
- **Responsive Design:** Immersive elements must adapt to mobile and desktop viewports.

## 3. Tech Stack
- **Framework:** Next.js 15 (App Router).
- **Styling:** Tailwind CSS + ShadCN UI.
- **Animations:** Framer Motion + Custom CSS Transitions.
- **Database:** Prepared for Firebase Firestore.
- **Auth:** Prepared for Firebase Auth.

## 4. User Stories
- *As a user, I want to feel like I am exploring a physical catalog so that learning feels like an adventure.*
- *As a user, I want to save interesting opportunities for later so that I don't lose track of them.*
- *As a user, I want to easily navigate to the actual course provider to begin my studies.*
